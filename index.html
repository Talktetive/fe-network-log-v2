Privacy Policy for FE Network LOG v2

Last updated: January 2026

1. Overview

FE Network LOG v2 is a browser extension designed for frontend debugging and QA testing. It records browser tab sessions and saves debugging data locally for bug analysis and reproduction.

2. Data Collection

The extension may collect the following data only during a user-initiated recording session:

  • Screen video of the selected browser tab
  • User interactions (mouse clicks, keyboard input, scrolling, navigation events)
  • Network request metadata (URLs, methods, headers, payloads, timing)
  • Console logs generated by the web page
  • Browser storage data related to the active tab (Cookies, Local Storage, Session Storage)
  • Technical metadata (URL, page title, browser version, user-agent, viewport size, IP and country)

3. Purpose of Data Usage

All collected data is used solely for debugging, bug reproduction, and QA analysis. The extension does not use data for advertising, analytics, or tracking.

4. Data Storage and Processing

  • All data is stored locally on the user’s device
  • Data is saved into a local .bugreport file
  • No data is uploaded to external servers

5. Data Sharing

FE Network LOG v2 does not sell, share, or transfer user data. Any sharing happens only if the user manually shares the generated file.

6. Remote Code

The extension does not load or execute remote code. All logic is bundled within the extension package.

7. User Control

  • Recording starts only after explicit user action
  • Users can stop recording at any time
  • Users fully control file saving and sharing

8. Data Security

Because all data remains on the local device, the risk of unauthorized access is minimized.

9. Changes to This Policy

This policy may be updated from time to time. Any changes will be reflected on this page.
