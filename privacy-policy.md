# Privacy Policy for FE Network LOG v2

**Last updated:** January 2026

## 1. Overview

FE Network LOG v2 is a browser extension designed for frontend debugging and QA testing.
The extension records browser tab sessions and saves debugging data locally for bug analysis and reproduction.

We respect user privacy and are committed to protecting it.

---

## 2. Data Collection

The extension may collect the following data **only during a user-initiated recording session**:

- Screen video of the selected browser tab
- User interactions (mouse clicks, keyboard input, scrolling, navigation events)
- Network request metadata (URLs, methods, headers, payloads, timing)
- Console logs generated by the web page
- Browser storage data related to the active tab:
  - Cookies
  - Local Storage
  - Session Storage
- Technical metadata:
  - Page URL and title
  - Browser name and version
  - User-Agent string
  - Viewport size
  - IP address and country (used only to enrich the local report)

---

## 3. Purpose of Data Usage

All collected data is used solely for:

- Debugging and reproducing frontend issues
- Creating bug reports for developers and QA teams

The extension does not use collected data for advertising, analytics, or tracking.

---

## 4. Data Storage and Processing

- All data is stored locally on the userâ€™s device
- Data is saved into a local `.bugreport` file
- No data is uploaded or transmitted to external servers

---

## 5. Data Sharing

FE Network LOG v2 does not sell, share, or transfer user data to third parties.

Users may choose to share the generated bug report file manually.

---

## 6. Remote Code

The extension does not load or execute remote code.
All logic is bundled within the extension package.

---

## 7. User Control

- Recording starts only after explicit user action
- Users can stop recording at any time
- Users control whether to save or share the report file

---

## 8. Data Security

Because all data remains on the local device, the risk of unauthorized access is minimized.
